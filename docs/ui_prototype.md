# UI 프로토타입 명세

> Claude.ai에서 HTML/JS로 제작된 프로토타입 기준.
> Spring Boot + JSP 전환 시 이 문서를 참고해 화면을 구현한다.

---

## 화면 목록

| 화면 ID | URL (예정) | 설명 | 접근 권한 |
|---|---|---|---|
| dashboard | `/dashboard` | 훈련사 메인 대시보드 | Trainer |
| surveys | `/surveys` | 문진 목록 | Trainer |
| survey-detail | `/surveys/{id}` | 문진 상세 + AI 분석 | Trainer |
| cases | `/cases` | 훈련 케이스 목록 | Trainer |
| case-detail | `/cases/{id}` | 케이스 상세 + 훈련일지 | Trainer |
| survey-link | `/survey-link` | 문진 링크 생성 | Trainer |
| owner-survey | `/survey/public/{token}` | 보호자 문진 작성 | Public |
| profile | `/profile` | 내 프로필 | Trainer |

## 화면별 상세

### 1. 대시보드 (`/dashboard`)

**상단 통계 카드 4개**
- 전체 고객 수 (훈련 케이스 총 수)
- 훈련 진행 중 (status = '훈련 중')
- 미확인 문진 (survey.status = '미확인')
- 이번 달 완료 (status = '완료', 이번 달 기준)

**최근 문진 접수 테이블**
- 컬럼: 보호자, 반려견, 상태
- 클릭 시 문진 상세로 이동

**훈련 진행 현황 테이블**
- 컬럼: 반려견 · 보호자, 상태(dot + 텍스트), 회차 수
- 클릭 시 케이스 상세로 이동

---

### 2. 사전 문진 목록 (`/surveys`)

**테이블 컬럼:** 보호자 / 반려견 / 견종·나이 / 문제행동(태그) / 접수일 / 상태 / AI분석 여부

**문진 상태값:** `미확인` | `확인 완료`

**행 클릭 시:** 페이지 하단에 상세 패널 슬라이드 인 (별도 페이지 이동 아님)

**문진 상세 패널 (2컬럼)**
- 좌측: 반려견 정보(이름/견종/나이/성별/중성화), 문제행동 내용
- 우측 상단: 보호자 정보(이름/접수일/상태)
- 우측 하단: AI 분석 박스
  - AI 미요청 상태: "AI 분석 요청" 버튼 표시
  - AI 완료 상태: 행동유형 / 위험도 / 요약 / 훈련 방향 표시
- 하단 액션: "훈련 케이스 생성" 버튼 (케이스 이미 존재하면 숨김) + "확인 완료 처리" 버튼

---

### 3. 훈련 케이스 목록 (`/cases`)

**상단:** "신규 케이스 생성" 버튼

**테이블 컬럼:** 반려견(견종·나이) / 보호자 / 상태 / 시작일 / 총 회차

**케이스 상태값:** `접수 완료` | `상담 중` | `훈련 중` | `완료`

**행 클릭 시:** 케이스 상세 패널 (아래 확장)

**케이스 상세**
- 헤더: 케이스명 + 상태 변경 드롭다운 (변경 즉시 목록 반영)
- 탭: [훈련 일지] / [일지 작성]
  - 훈련 일지 탭: 회차별 카드 (목표 / 내용 / 개선사항 / 수정·삭제 버튼)
  - 일지 작성 탭: 훈련 목표 / 훈련 내용 / 개선사항 입력 폼

---

### 4. 케이스 생성 모달

**문진에서 진입 시:** 보호자명 / 반려견명 / 견종 자동 입력 (readonly)

**케이스 목록에서 직접 생성 시:** 빈 폼 (직접 입력)

**필드:** 보호자 / 반려견 이름 / 견종 / 초기 상태(접수 완료|상담 중) / 메모(선택)

---

### 5. 문진 링크 생성 (`/survey-link`)

**입력:** 보호자 이름(필수) / 반려견 이름(선택)

**생성 결과:** 토큰 URL 표시 + 복사 버튼 + "보호자 문진 폼 미리보기" 버튼

**토큰 URL 형식:** `http://[host]/survey/public/{UUID_token}`

---

### 6. 보호자 문진 폼 (`/survey/public/{token}`)

- 사이드바/로그인 없이 단독 접근 가능
- 상단: "PetTrainer Pro · [훈련사명] 트레이너 · 보호자용 문진" 표시
- URL 파라미터로 보호자명/반려견명 사전 입력 가능

**입력 섹션 1 — 보호자 정보**
- 보호자 이름(필수) / 연락처

**입력 섹션 2 — 반려견 정보**
- 반려견 이름(필수) / 견종(필수) / 나이 / 성별(select) / 중성화 여부(select)

**입력 섹션 3 — 문제행동**
- 체크박스: 분리불안 / 짖음 / 공격성 / 배변 문제 / 산책 거부 / 물건 파괴 / 기타
- 상세 설명(textarea, 필수)
- 훈련 목표(textarea, 선택)

**제출 후:** 폼 숨김 + 성공 메시지 배너 표시

## UX 결정사항 (구현 시 반드시 반영)

1. **문진 → 케이스 생성**: 케이스 생성 모달에 문진의 보호자/반려견/견종 자동 주입. 케이스가 이미 존재하는 문진은 "케이스 생성" 버튼 숨김.
2. **AI 분석**: 훈련사가 버튼을 직접 눌러야 요청됨. 자동 실행 아님.
3. **상태 변경**: 드롭다운 변경 즉시 목록 테이블 배지도 동기화.
4. **훈련일지 삭제**: confirm 다이얼로그 후 삭제.
5. **훈련일지 수정**: 모달 팝업으로 처리.
6. **보호자 폼 사전 입력**: 링크 생성 시 입력한 보호자/반려견 이름을 URL 쿼리 파라미터로 전달해 폼에 prefill.
7. **문진 상세**: 별도 페이지 이동 없이 목록 하단에 패널 확장.
